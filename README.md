## Local services

```bash
docker compose up -d postgres qdrant minio
cargo run -p queria-api
cargo run -p queria-worker
```

Copy `.env.example` into the environment used by both binaries and replace
`QUERIA_SETUP_TOKEN`. The API applies bundled migrations and seeds the
`fjulian-me` source registry. The worker requires `git` and TruffleHog 3.x on
`PATH`.

## Git ingestion API

All ingestion endpoints require the admin session cookie.

```text
POST /api/v1/sources/{source_document_id}/ingest
GET  /api/v1/ingestion-jobs?status=running&limit=50
GET  /api/v1/ingestion-jobs/{job_id}
POST /api/v1/ingestion-jobs/{job_id}/retry
POST /api/v1/ingestion-jobs/{job_id}/cancel
```

`retry` accepts only `failed` or `cancelled` jobs. `cancel` immediately closes
a queued job and requests cooperative cancellation for a running job. A source
can have only one queued/running `git_ingestion` job; duplicate triggers return
the active job.

The worker claims jobs with `FOR UPDATE SKIP LOCKED`. Terminal job results use:

```json
{
  "indexed_files": 12,
  "unchanged_files": 30,
  "deprecated_files": 2,
  "knowledge_items": 48,
  "chunks": 53
}
```

## Ingestion controls

- Repository paths are canonicalized and must be below
  `QUERIA_GIT_ALLOWED_ROOTS`.
- SSH URIs must match both `QUERIA_GIT_ALLOWED_SSH_HOSTS` and
  `QUERIA_GIT_ALLOWED_SSH_REPOSITORIES`.
- TruffleHog runs before files are parsed. Its include/exclude files match the
  indexable file policy; any eligible finding or scan error fails the job.
- Supported MVP inputs are Markdown/MDX, Astro, TypeScript/JavaScript,
  JSON/JSONC, YAML, and TOML.
- Chunk boundaries are line-based, deterministic, 1-based, and inclusive.
  `QUERIA_GIT_CHUNK_MAX_LINES` and `QUERIA_GIT_CHUNK_OVERLAP_LINES` are
  parameterized.
- Unchanged files keep their knowledge. Changed files supersede old generated
  knowledge. Deleted files become inactive, their knowledge becomes
  `deprecated`, and their chunks are removed from retrieval.

## Trusted auto-approval

Git auto-approval requires all of these conditions:

1. The root source metadata contains `"trusted_auto_approve": true`.
2. Path and optional SSH URI allowlists pass.
3. TruffleHog completes with no findings or scan errors.
4. The knowledge is generated by the Git worker pipeline.

The approval row, knowledge metadata, chunk metadata, and audit log record the
pipeline identity and commit SHA. Non-trusted Git roots create pending
proposals without retrieval chunks. Agent `propose_memory` remains manual and
cannot enter this auto-approval path.
